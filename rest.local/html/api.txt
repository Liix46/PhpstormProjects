Разработка Api

Работу API обеспечивают скрипты PHP, рекомендуется
обращаються к ним через маршрутизатор (.htaccess)
Это позволит а) скрыть расширение (технологию),
б) убедиться в работоспособности сервера (если не
сработала маршрутизация, то проблемы с сервером)

Разделение по методам запроса происходит из анализа
$_SERVER[ 'REQUEST_METHOD']

Передача данных:
данные из URL (Query) попадают в массив $_GET независимо
от метода запроса.
Данные из тела попадают в массив $_POST, но, только если
заголрвок Content-Type соответсвует html-форме.Иначе,
доступ к телу пакета запроса предостовляет специяальный
поток (файл) "php://input". С ним возможны любые файловые
операции чтения, например, file_get_contents( "php://input" ),
возвращающий весь контент в виде строки.
Ответные данные передаються выводящими операциями echo или print,
а также средства разработчика var_dump.
Статус-код ответа устанавливается http_response_code(412);
заголовки
header ( "Key: value" ) ;

Основные параметры запроса и режимов работы собираються в массиве
$_SERVER[ ], его состав зависит от версии Apache, PHP, ОС.
Данные лучше уточнить при конкретной реализации.
